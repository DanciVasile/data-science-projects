Priority order (estimated effort in parentheses):

  P2  = medium effort, strong payoff     (3-4 h total)

────────────────────────────────────────────────────────────────────────────────
A. MODEL PERFORMANCE                                                     [P1-P2]
────────────────────────────────────────────────────────────────────────────────
1. More feature engineering
   - Interaction terms (grade × dti, term × int_rate)
   - Polynomial features for top-correlated numerics
   - Bin annual_inc into quantiles (deciles) instead of raw values

2. Target encoding for high-cardinality categoricals
   - sub_grade, zip_code, addr_state
   - Use category_encoders.TargetEncoder with proper CV folds to avoid leakage
   - Wrap in a sklearn Pipeline so encoding is fit on train only

3. LightGBM                                                                 [P2]
   - Often outperforms XGBoost on tabular data
   - Handles categoricals natively (no need for one-hot encoding)
   - Faster training on large datasets

4. Stacking ensemble                                                         [P3]
   - Blend LR + RF + XGBoost predictions as meta-features
   - Train a meta-learner (e.g., Logistic Regression) on out-of-fold predictions
   - sklearn.ensemble.StackingClassifier makes this easy

5. Threshold optimization                                                    [P2]
   - Tune the classification threshold on the Precision-Recall curve
   - Instead of using the default 0.5, find the threshold that maximizes F1
   - Use the validation set (not test set) for threshold selection

6. Close the overfit gap (train AUC 0.80 vs test AUC 0.73)                  [P2]
   - Increase XGBoost regularisation: reg_alpha, reg_lambda, min_child_weight
   - Reduce max_depth range from [3,10] to [3,7]
   - Try colsample_bytree ∈ [0.5, 0.8]
   - Add early_stopping_rounds to prevent over-training
